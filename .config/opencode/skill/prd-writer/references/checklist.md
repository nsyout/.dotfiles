# PRD Completeness Checklist

Use this checklist before writing `workflow/PRD.md` and again before final handoff.

## Input Quality

- [ ] At least one clarification round completed
- [ ] Source materials ingested (notes/docs/links)
- [ ] Confirmed Facts list created
- [ ] Assumptions list created
- [ ] Open Questions list created
- [ ] Contradictions identified and flagged

## Section Completeness

- [ ] Title / Problem Statement
- [ ] Users / Stakeholders
- [ ] Goals and Non-Goals
- [ ] Scope (In / Out)
- [ ] Constraints (Technical, Org, Timeline)
- [ ] Functional Requirements
- [ ] Non-Functional Requirements
- [ ] Acceptance Criteria (Testable)
- [ ] Success Metrics
- [ ] Risks and Mitigations
- [ ] Dependencies
- [ ] Rollout / Milestones (High Level)
- [ ] Open Questions
- [ ] Decision Log (Initial)
- [ ] Next Step

## Quality Gates

- [ ] No premature implementation details dominate the document
- [ ] Requirements are unambiguous and use measurable language
- [ ] Acceptance criteria are pass/fail testable
- [ ] Success metrics include baseline/target/time window (or explicit TODO)
- [ ] Scope boundaries are explicit and non-overlapping
- [ ] Risks include concrete mitigations
- [ ] Dependencies have owners/status
- [ ] Any missing critical info marked as `TODO:`
- [ ] Deterministic heading order preserved

## Final Output Checks

- [ ] File written to `workflow/PRD.md`
- [ ] Response confirms file path
- [ ] Response includes handoff to `spec-planner`
- [ ] No task breakdown included unless user explicitly asked
- [ ] Optional appendix included only if user asked
